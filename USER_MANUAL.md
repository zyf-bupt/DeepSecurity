# SecTrace 使用说明书

## 基于大模型的攻击行为全方位检测·捕获·溯源系统

---

## 一、系统概述

SecTrace 是一个基于大语言模型（LLM）的攻击行为检测、捕获与溯源系统，实现了从 **检测（Detection）→ 捕获（Capture）→ 溯源（Attribution）** 的端到端攻击行为分析链路。

### 核心技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 检测引擎 | LLM + RAG | Qwen-Flash 模型 + TF-IDF 语义检索增强 |
| 知识图谱 | Neo4j (可选) | 进程/文件/网络/IP/用户实体关系图 |
| 捕获框架 | Multi-Agent | 4个协作Agent：收集→构建→验证→裁决 |
| 归因引擎 | TAA-EPLMR | 证据路径推理 + Jaccard TTP匹配 |
| 威胁情报 | MITRE ATT&CK | 24项技术 + 5个APT组织档案 |
| 前端框架 | Flask + 原生JS | 明亮主题 · 侧边栏导航 · Canvas可视化 |

### 系统架构图

```
场景管理 → 生成攻击数据 (3类数据源)
    ↓
检测引擎 → RAG检索 + 规则匹配 + LLM深度分析 → 告警分级(高/中/低)
    ↓
捕获框架 → 4-Agent协作 → 攻击链重构 → 证据固化(SHA-256哈希链)
    ↓
归因引擎 → TTP匹配 → 威胁情报融合 → 攻击者画像 → 自动报告
```

## 三、网页各板块功能详解

### 3.1 系统首页 (`/`)
系统概览页面，展示系统功能总览、核心模块介绍和快速入口。包含检测/捕获/溯源的流程说明。

### 3.2 仪表盘 (`/dashboard`)
全局数据看板，展示四大KPI指标：
- **主机日志数量**：Windows Event Log / Linux Auditd 日志条目
- **进程行为数量**：Sysmon/Falco 采集的进程创建、文件操作、网络连接事件
- **网络流量数量**：TCP/UDP/HTTP/DNS/ICMP 流量记录
- **攻击报告数量**：已生成的攻击分析报告

同时展示数据新鲜度（最新数据时间戳）和最近的攻击报告摘要列表。

### 3.3 场景管理 (`/scenario`) ★核心功能

**功能说明**：生成真实的攻击事件数据，驱动整个检测→捕获→溯源管线。

**两个内置场景**：

#### 场景一：APT全链条攻击 (Lazarus Group 模式)
模拟朝鲜 Lazarus Group 风格的完整 APT 攻击链，包含 8 个阶段：
1. **侦察** — 端口扫描、漏洞探测 (nmap/SYN scan)
2. **初始入侵** — 公网Web应用漏洞利用，wget下载投递工具 (T1190/T1105)
3. **执行与持久化** — Payload执行、Webshell部署、Cron定时任务 (T1059/T1547/T1053)
4. **凭据窃取** — /etc/shadow敏感文件读取、SSH暴力破解 (T1003/T1110)
5. **横向移动** — SSH登录核心服务器、WMI攻击域控 (T1021/T1047)
6. **C2通信** — DNS隧道(高熵子域名) + ICMP隧道(大载荷Ping) (T1071/T1095)
7. **数据外传** — 加密数据库文件分段HTTPS外传到C2 (T1048)
8. **痕迹清除** — 日志清理、命令历史删除 (T1070)

**涉及攻击路径**：外部C2(45.33.22.11) → DMZ Web(192.168.86.10) → 跳板机(192.168.86.132) → Linux核心(192.168.86.130) → Windows域控(192.168.86.131)

#### 场景二：AI智能体滥用攻击 (GhostInTheMachine 模式)
模拟攻击者利用 Claude Code / OpenCode 等 LLM Agent 发起的自动化攻击，包含 6 个阶段：
1. **AI Agent接入** — LLM代理在工作站启动，执行系统侦察(whoami/ifconfig/find)
2. **AI载荷生成** — LLM生成恶意Python脚本（含LLM典型特征：详细注释、try-except-pass）
3. **工具链滥用** — AI Agent组合curl/wget/python/bash进行多工具攻击
4. **横向渗透** — AI探索内网拓扑，多路径切换试错（SSH→WMI→SMB）
5. **AI供应链攻击** — 攻击AI模型注册表，尝试模型污染
6. **数据外传** — 通过AI基础设施C2通道外传数据

**检测关键点**：
- AI Agent 行为具有 explore-exploit 循环特征
- 命令间隔远小于人类操作（无人工思考停顿）
- 代码包含 LLM 典型特征（详细注释、逻辑冗余）
- 试错路径切换（一个方法失败快速切换到另一个）

### 3.4 攻击检测 (`/detection`) ★核心功能

**功能说明**：基于大模型的攻击行为自动化识别，是整个系统的第一环。

**检测流程**：
1. **多源数据接入**：
   - 主机行为数据（进程创建 process_create、文件读写 file_read/write、网络连接 network_connection）
   - 网络流量数据（网络流 network_flow、DNS查询、ICMP包）
   - 主机日志数据（用户登录 user_logon、登录失败 user_logon_failed、日志清除 log_clear）

2. **三层检测架构**：
   - **规则引擎层（快速通道）**：9条内置规则，覆盖敏感文件访问、隐蔽信道、WMI执行、数据外传、AI Agent模式等
   - **RAG增强层（语义理解）**：TF-IDF向量检索 MITRE ATT&CK 知识库（37篇安全文档），余弦相似度匹配
   - **LLM深度分析层（智能判断）**：Qwen-Flash 大模型对高可疑事件进行语义级分析

3. **告警分级（高/中/低）**：
   - 高危：C2通信、数据外传、凭据转储、横向移动
   - 中危：命令执行、WMI执行、端口扫描、持久化
   - 低危：登录失败、信息探测

4. **MITRE ATT&CK 映射**：每条告警自动映射到 MITRE ATT&CK 框架的战术/技术ID

**操作方式**：
- 先在「场景管理」启动攻击场景
- 回到「攻击检测」页面，点击「执行全管线分析」
- 查看实时告警流、战术分布统计
- 使用 RAG 知识库检索搜索攻击模式

### 3.5 溯源归因 (`/attribution`) ★核心功能

**功能说明**：对捕获的攻击行为进行攻击者归因分析，生成攻击者画像。

**归因流程**：
1. **TTP 提取与匹配**：从攻击链中提取 TTP 列表，使用 Jaccard 相似度与已知 APT 组织进行匹配
2. **威胁情报融合**：整合 MITRE ATT&CK 知识库 + APT 组织档案（包含 Lazarus、APT28、APT29、APT41 及新兴 AI 威胁行为者）
3. **行为画像分析**：技能水平评估（高级/中级/初级）、攻击复杂度、主要目的、行动安全
4. **基础设施分析**：C2节点数量、内外网IP分布、基础设施复杂度
5. **证据路径推理**：构建「入口识别→TTP提取→IOC提取→归因推理」的证据链
6. **综合归因**：加权综合得出归因类型和置信度
   - Known APT（TTP相似度 ≥ 0.3）
   - Suspected Group（TTP相似度 ≥ 0.1）
   - Unknown Cluster（低匹配或无匹配）

**攻击者画像维度**：
| 维度 | 内容 |
|------|------|
| 身份特征 | 疑似组织、来源国家、归因置信度 |
| 能力评估 | 技能水平、攻击复杂度、战术范围、使用工具、AI能力 |
| 动机评估 | 主要目的、可能动机、影响严重度 |
| 行为模式 | 初始访问方式、横向移动方式、C2方式、外传方式 |
| IOC画像 | 基础设施类型、C2 IP列表、内部跳板、域名 |
| 威胁等级 | 0-10分评分 + CRITICAL/HIGH/MEDIUM/LOW 等级 |

### 3.6 攻击链分析 (`/attack`)

**功能说明**：基于 Neo4j 图数据库展示攻击事件之间的因果关联关系，支持宏观 ATT&CK 链和微观实体拓扑两个视图。

### 3.7 溯源分析 (`/traceback`)

**功能说明**：深度溯源分析页面，展示高置信度告警列表，支持选中告警后查看完整攻击路径和 AI 生成的溯源报告。

### 3.8 日志分析 (`/logs`)

**功能说明**：Windows Event Log / Linux Auditd 日志的检索与查看，支持登录会话追踪（LogonTracer）可视化。

### 3.9 行为监控 (`/behavior`)

**功能说明**：主机行为监控面板，展示 Sysmon/Falco 采集的进程创建、文件操作、注册表修改等终端行为。

### 3.10 流量分析 (`/traffic`)

**功能说明**：网络流量分析面板，展示 TCP/UDP/HTTP/DNS/ICMP 流量记录，支持在线抓包和离线 pcap 导入。

### 3.11 网络拓扑 (`/scenario/network`)

**功能说明**：Canvas 动态拓扑图，可视化展示 6 节点企业网络：
- 外部威胁区：C2 服务器 (45.33.22.11)
- DMZ 隔离区 (低安全)：Web 服务器 (192.168.86.10)
- 内网核心区 (中安全)：Linux 应用服务器 (192.168.86.130)、跳板机 (192.168.86.132)
- 内网高安全区：Windows 域控制器 (192.168.86.131)
- 安全管理区：SOC 分析平台 (192.168.86.200)

点击节点查看详情，已攻陷节点显示红色光晕。

## 八、常用操作流程

### 完整分析流程
1. **首页** → 了解系统概况
2. **场景管理** → 点击「场景一」或「场景二」的「启动」
3. **攻击检测** → 点击「执行全管线分析」
4. **溯源归因** → 查看归因报告和攻击者画像
5. **网络拓扑** → 查看攻击影响范围和节点状态

### 单独功能使用
- **RAG搜索**：在检测页面的搜索框输入 TTP ID（如 T1071）或关键词（如 DNS隧道）
- **查看告警**：检测页面左侧告警流，按严重程度颜色区分
- **查看报告**：归因页面点击「生成最新报告」
- **清除数据**：场景管理页面「清除全部」
