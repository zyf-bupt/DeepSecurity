# 基于 Dumpcap 的流量采集与离线分析教程

## 一、文档说明

本教程适配**离线抓包 + 事后解析入库**的流量分析场景，基于 Wireshark 自带的`dumpcap`工具实现轻量化抓包：抓包过程仅保存原始 pcap 文件、不做实时分析，停止抓包后统一进行解析、威胁检测并入库，完美匹配项目 “检测 - 捕获 - 溯源” 的流量侧需求。

适用环境：Windows 系统，Wireshark 安装路径 例： `E:\wireshark`

## 二、环境准备与验证

### 2.1 工具确认

```

cd /d E:\wireshark

dumpcap.exe -v
```

## 三、核心参数说明

完全匹配项目配置项，对应 dumpcap 命令参数：

|   项目配置项    |             命令参数             |                           详细说明                           | 必填性 |
| :-------------: | :------------------------------: | :----------------------------------------------------------: | :----: |
|  iface（网卡）  |       `-i <网卡设备路径>`        |        指定抓包的网卡，填完整的`\Device\NPF_xxx`路径         |  必填  |
| BPF（过滤规则） |         `-f "<BPF规则>"`         |    抓包时过滤流量，减少 pcap 文件体积；留空则抓取全部流量    |  可选  |
|    保存文件     |   `-w <保存路径\文件名.pcap>`    |       指定抓包原始文件的保存路径，用于后续离线解析入库       |  必填  |
|  自动停止条件   | `-a duration:秒数` / `-c 包数量` |      按时长 / 包数量自动停止抓包，也可手动`Ctrl+C`停止       |  可选  |
|  dumpcap_path   |                -                 |    工具本身路径，本环境固定为 `E:\wireshark\dumpcap.exe`     |  固定  |
|    host_name    |                -                 | 入库标记字段，不在抓包命令中体现，解析入库时写入数据库，用于标记采集点 |  可选  |
|    分析开关     |                -                 | 离线解析时启用，启用后生成`detections`/`anomalies`等结构化检测字段 |  可选  |

## 四、抓包操作步骤

### 4.1 基础抓包（无过滤）

1.以**管理员身份**打开 CMD，进入 Wireshark 目录：

```
cd /d E:\wireshark
```

2.创建 pcap 文件保存目录（避免路径不存在报错）：

```
mkdir E:\pcap_log
```

3.启动抓包（示例：抓取 WLAN 无线网卡的全部流量）：

```
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -w E:\pcap_log\test.pcap
```

4.停止抓包：

在 CMD 窗口按下 Ctrl + C，工具会自动收尾并完整保存 pcap 文件。

### 4.2 带 BPF 过滤的抓包

添加 BPF 规则，只抓取符合条件的流量，大幅减小文件体积，示例：

```
# 示例1：只抓取192.168.86.0/24整个网段的流量
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -f "net 192.168.86.0/24" -w E:\pcap_log\net_test.pcap

# 示例2：只抓取和192.168.86.131单台主机相关的流量
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -f "host 192.168.86.131" -w E:\pcap_log\host_test.pcap

# 示例3：只抓取ICMP协议流量
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -f "icmp" -w E:\pcap_log\icmp_test.pcap
```

### 4.3 自动停止抓包

设置自动停止条件，无需手动干预：

```
# 示例：抓取300秒（5分钟）后自动停止
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -a duration:300 -w E:\pcap_log\auto_stop.pcap

# 示例：抓取1000个数据包后自动停止
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -c 1000 -w E:\pcap_log\count_stop.pcap
```

### 4.4 大文件自动分割

长时间抓包时，按大小 / 时间自动分割 pcap 文件，避免单文件过大无法解析：

```
# 示例：每100MB生成一个新文件，最多保留10个文件
dumpcap.exe -i \Device\NPF_{C362C346-3200-459A-AE7C-C8A1E59E7725} -b filesize:102400 -b files:10 -w E:\pcap_log\split.pcap
```

## 五、常用 BPF 过滤规则速查

BPF（伯克利包过滤器）是抓包层的标准过滤语法，支持按主机、网段、端口、协议自由组合：

|   规则类型   |                        示例                        |                 说明                  |
| :----------: | :------------------------------------------------: | :-----------------------------------: |
|   主机过滤   |                `host 192.168.1.100`                |    抓取源或目的为该 IP 的所有流量     |
|   网段过滤   |                `net 192.168.1.0/24`                |          抓取整个 C 段的流量          |
| 源 / 目的 IP | `src host 192.168.1.100`  `dst host 192.168.1.100` |      仅过滤源 IP 或仅过滤目的 IP      |
|   端口过滤   |      `port 80`  `tcp port 443`  `udp port 53`      | 抓取指定端口流量，可指定 TCP/UDP 协议 |
|   协议过滤   |       `icmp` / `tcp` / `udp` / `arp` / `dns`       |        抓取指定协议的全部流量         |
|   组合规则   |        `host 192.168.1.100 and tcp port 80`        | 同时满足多个条件（`and`/`or`/`not`）  |
|   排除规则   |                  `not port 3389`                   |  排除远程桌面等无关流量，减小包体积   |

> 靶场实战常用组合：
>
> ```
> host 192.168.86.131 and not port 3389
> ```

## 六、离线解析与分析入库

抓包停止后，对 pcap 文件进行统一解析、威胁检测，并将结果写入数据库，匹配项目 “停止并导入入库” 的核心需求。

### 6.1 基础结构化解析

使用 Wireshark 自带的`tshark`命令行工具，将 pcap 解析为结构化数据，方便入库：

```
# 进入Wireshark目录
cd /d E:\wireshark

# 示例：解析pcap，输出时间戳、五元组、包长等核心字段，保存为CSV格式
tshark.exe -r E:\pcap_log\test.pcap -T fields -e frame.time -e ip.src -e ip.dst -e tcp.srcport -e tcp.dstport -e udp.srcport -e udp.dstport -e ip.proto -e frame.len -E header=y -E separator=, > E:\pcap_log\flow_result.csv
```

### 6.2 启用威胁检测（分析开关开启）

开启分析开关后，生成`detections`、`anomalies`等结构化检测字段：

1.**端口扫描检测**：统计同一源 IP 短时间内访问大量目的端口的行为

```
tshark.exe -r E:\pcap_log\test.pcap -z conv,tcp
```

2.**隐蔽信道检测**：识别 DNS 隧道、ICMP 隧道、非标准端口协议等异常流量

3.**攻击特征匹配**：结合 Suricata 规则库，匹配 pcap 中的攻击载荷与行为

### 6.3 入库标记（host_name）

解析完成写入数据库时，添加`host_name`字段标记采集点：

- 示例值：`WIN-TEST-PC01`
- 作用：区分不同采集端的流量数据，便于攻击溯源时与 Sysmon 主机行为日志关联

## 七、靶场环境推荐配置

结合项目 “不少于 5 台主机、DMZ / 内网分层、高低安全域” 的靶场需求，推荐抓包配置：

1. **网卡选择**：VMnet8（NAT 模式虚拟网卡），可抓取所有虚拟机节点的横向移动、南北向流量
2. **BPF 规则**：`net 192.168.86.0/24`，仅采集靶场网段流量，过滤公网无关包
3. **保存策略**：按 100MB 自动分割文件，保留 20 个文件，避免磁盘占满
4. 完整命令：

```
dumpcap.exe -i \Device\NPF_{EBA91C13-0A6F-4963-95DE-E408FF6CB55C} -f "net 192.168.86.0/24" -b filesize:102400 -b files:20 -w E:\pcap_log\target_env.pcap
```

## 八、常见问题排查

### 8.1 抓包提示 “没有权限 / 打开设备失败”

解决：必须以**管理员身份**运行 CMD，普通用户权限无法读取网卡底层流量。

### 8.2 网卡名称中文乱码

解决：不影响功能使用，直接复制`\Device\NPF_xxx`完整路径即可；可通过括号后的备注名对应网卡。

### 8.3 抓包全程没有数据

排查步骤：

1. 确认网卡选择正确，例如用 WiFi 上网却选了有线网卡
2. 确认 BPF 规则没有过滤掉所有流量，可先去掉`-f`参数测试
3. 手动产生流量验证，比如 ping 目标 IP、访问网页

### 8.4 pcap 文件体积过大

优化方案：

1. 添加 BPF 过滤规则，只采集关注的网段 / 端口 / 协议
2. 使用`-b`参数设置自动分割文件
3. 缩短单次抓包时长，分批次采集

## 九、与主机行为监控联动说明

本流量采集模块可与 Sysmon 主机行为监控模块配合，实现 “主机 + 网络” 全维度数据采集，支撑攻击链重构与溯源：

1. 同一台主机同时运行客户端行为 Agent 和流量抓包
2. 数据入库时通过`host_name`和时间戳做关联
3. 攻击溯源时可同时查看进程创建、文件修改、注册表变更等主机行为，和网络连接、载荷传输、C2 通信等流量行为，形成完整攻击链路
