**2\. 角色定义与功能 (Role Definitions)**

在答辩时，请按以下方式介绍这 5 个节点：

1. **外部 C2 服务器 (Node 1\)**：  
   * **定义**：模拟 APT 组织（如 Lazarus）的外部控制设施。  
   * **作用**：作为攻击链的**终点**（数据回传目的地）和**起点**（恶意指令下发源）。它是“身份溯源”的关键线索来源（IP、域名、指纹）。  
2. **攻击跳板机 / Patient Zero (Node 2\)**：  
   * **定义**：内网中第一台失陷的主机（运行我们 Python 脚本的机器）。  
   * **作用**：模拟已被黑客控制的内网节点。它是**攻击源头**，负责发起扫描、爆破和横向移动。  
3. **受害靶机群 (Node 3 & 4\)**：  
   * **定义**：承载业务的 Linux 和 Windows 服务器。  
   * **作用**：**攻击承受者**。它们负责产生被攻击的痕迹（如 Auth.log 失败记录、Sysmon 进程创建日志、异常文件变动），供 Agent 采集。  
4. **溯源分析系统服务端 (Node 5 \- 核心)**：  
   * **定义**：**全知全能的上帝视角**，运行着你项目代码（little\_term）的服务器。  
   * **作用**：  
     * **采集层**：接收 Node 2,3,4 的日志和全网流量。  
     * **计算层**：运行 APT\_Analysis\_Engine.py 和 covert\_channel\_detector.py 等算法。  
     * **展示层**：绘制出“攻击者是谁”以及“他走了哪条路”。

**3\. 攻击实现 vs. 预期检测效果 (Attack vs. Detection)**

这是最关键的部分，解释了**“我做了什么攻击”** 以及**“我的系统能检测出什么”**。请将此表印在脑海里：

| :---- | :---- | :---- |
| **行为**：wget .../dropper.sh **行为**：chmod \+x; ./dropper.sh | **T1105** (工具传输) **T1059** (命令执行) | **\[流量分析\]**：检测到 HTTP 请求下载可疑脚本，User-Agent 匹配 Lazarus 特征。 **\[主机行为\]**：Falco 规则触发“Shell 脚本启动”或“非预期进程网络连接”。 |
| **行为**：多线程 Socket 连接端口 22/445 | **T1046** (网络服务扫描) | **\[流量分析\]**：anomaly\_detector.py 触发“端口扫描”或“内网地址扫描”告警（短时间内同一源 IP 连接大量目标 IP）。 |
