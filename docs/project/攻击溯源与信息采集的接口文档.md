### 主机日志提取函数返回结果：list(列表类型):

```python
        [
        {
        "data_source": "host_log",
        "timestamp": "2023-10-27T10:00:01Z", // 时间序列对齐后的标准时间
        "host_ip": "192.168.1.100",
        "event_type": "login_success",        // 归一化后的事件类型
        "raw_id": "4624",                     // 原始Windows事件ID
        "entities": {                         // 提取的关键实体
        "user": "administrator",
        "src_ip": "192.168.1.5",
        "src_port":
        "session_id": "0x12345"
        "service_name":
		"file_path":                         //(执行文件路径)
        },
        "description": "User administrator logged in from 192.168.1.5",
        }
        ]
```

| **event_type (枚举值)**       | **对应原始事件 (示例)**                     | **ATT&CK 映射方向**                         | **题目功能对应**       |     |
| -------------------------- | ----------------------------------- | --------------------------------------- | ---------------- | --- |
| **`user_logon`**           | Win: 4624, Linux: Accepted password | T1078 (Valid Accounts)                  | 登录会话重建 / 身份溯源    |     |
| **`user_logoff`**          | Win: 4634/4647                      | -                                       | 确定会话结束时间         |     |
| **`user_logon_failed`**    | Win: 4625, Linux: Failed password   | T1110 (Brute Force)                     | 检测暴力破解           |     |
| **`process_creation_log`** | Win: 4688 (Security Log)            | T1059 (Cmd & Scripting)                 | 进程链分析 (日志侧备份)    |     |
| **`service_install`**      | Win: 7045/4697                      | T1543 (Create or Modify System Process) | 权限维持检测           |     |
| **`account_created`**      | Win: 4720                           | T1136 (Create Account)                  | 攻击者留后门           |     |
| **`group_membership_add`** | Win: 4728/4732/4756                 | T1098 (Account Manipulation)            | 权限提升检测 (加Admin组) |     |
| **`log_clear`**            | Win: 1102, Linux: rm /var/log       | T1070 (Indicator Removal)               | 反取证检测            |     |

##### 键值对详细解释：

- **`data_source`: "host_log"**
  
    - **含义**：标识这是一条来自“日志文件”的数据，区别于行为监控或流量分析。
      
    - **题目对应**：多源数据融合的分类标签。
    
- **`timestamp`**
  
    - **含义**：ISO 8601 格式的 UTC 时间。
      
    - **题目对应**：**“日志的时间序列对齐”**。无论原始日志是本地时间还是其他时区，这里必须统一转换为标准时间，确保多台主机的时间轴能对得上。
    
- **`host_ip`**
  
    - **含义**：产生日志的主机 IP 地址。
      
    - **作用**：在图数据库中对应 `IP` 节点，是溯源的起点。
    
- **`event_type`**
  
    - **含义**：**归一化后的事件名称**。例如，Windows 的 `4624` 和 Linux 的 `Accepted password` 都会被映射为 `user_logon`。
      
    - **题目对应**：**“日志范式解析”**。将不同格式转换为统一范式，方便后端逻辑统一处理（如检测暴力破解时，只需查 `user_logon_failed` 而不用管是 Win 还是 Linux）。
    
- **`raw_id`**
  
    - **含义**：原始日志中的 ID（如 Windows Event ID 4624）。
      
    - **作用**：用于保留证据，方便取证时回溯原始日志文件。
    
- **`entities` (字典对象)**
  
    - **含义**：从日志文本中提取出的核心信息。
      
    - **题目对应**：**“关键信息提取”**。
      
        - `user`: 操作者账号（用于 **“身份溯源”**）。
          
        - `src_ip`: 攻击源 IP（用于 **“登录会话重建”**）。
          
        - `session_id`: 登录会话的唯一标识（用于关联同一会话下的所有操作）。

### 主机行为异常监测函数返回结果（list）

```python
        [
        {
        "data_source": "host_behavior",
        "timestamp": "2023-10-27T10:05:00Z",
        "host_ip": "192.168.1.100",
        "event_type": "process_create",
        "action": "execution",
        "entities": {
        "process_name": "powershell.exe",
        "pid": 4096,
        "parent_process": "cmd.exe",
        "parent_pid": 3012,
        "command_line": "powershell.exe -enc AAB..." // 异常参数
        "hash": "a1b2c3d4...",
        "user": "SYSTEM",                   // [新增] 执行用户，用于分析提权
        "file_path": "C:\\Windows\\Temp\\malware.exe"
        "listen_ports": [80, 443]          // [新增] 该进程启动时或快照时绑定的端口
        // [新增，场景B: 注册表相关] (registry_set_value) 
        "registry_key": "HKLM\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run", 
        "registry_value_name": "UpdateService", 
        "registry_value_data": "C:\\Windows\\Temp\\malware.exe",
        // [新增，场景C: 网络相关] (network_connection) 
        "src_ip": "192.168.1.100", "src_port": 50123, 
        "dst_ip": "142.250.1.1", 
        "dst_port": 443, "protocol": "TCP" 
        },
        "behavior_features": {
        "is_abnormal_parent": true,        // 进程父子关系异常
        "has_memory_injection": false
        },
        }
        ]

```
event_type范围：

| **event_type (枚举值)**     | **触发场景**                  | **ATT&CK 映射方向**                    | **图数据库关联 (Edge)**              |
| ------------------------ | ------------------------- | ---------------------------------- | ------------------------------ |
| **`process_create`**     | 进程启动                      | T1059 (Execution)                  | Process -> `Spawn` -> Process  |
| **`process_terminate`**  | 进程结束                      | -                                  | (用于闭合进程生命周期)                   |
| **`file_create`**        | 落地文件 (Webshell/木马)        | T1105 (Ingress Tool Transfer)      | Process -> `Write` -> File     |
| **`file_modify`**        | 修改文件 (如篡改配置)              | T1070.004 (File Deletion/Mod)      | Process -> `Write` -> File     |
| **`file_delete`**        | 删文件 (清理痕迹)                | T1070.004                          | Process -> `Delete` -> File    |
| **`file_read`**          | 读取敏感文件 (如 shadow)         | T1005 (Data from Local System)     | Process -> `Read` -> File      |
| **`registry_set_value`** | 修改注册表 (启动项)               | T1547 (Boot or Logon Autostart)    | Process -> `Write` -> Registry |
| **`image_load`**         | 加载 DLL                    | T1574 (Hijack Execution Flow)      | Process -> `Load` -> File      |
| **`process_injection`**  | 内存操作 (CreateRemoteThread) | T1055 (Process Injection)          | Process -> `Inject` -> Process |
| **`network_connection`** | 进程发起网络连接                  | T1071 (Application Layer Protocol) | Process -> `Connect` -> IP     |



##### 键值对详细解释：

- **`event_type`**
  
    - **含义**：具体的行为动作类型，如 `process_create`（进程创建）、`file_create`（文件创建）、`image_load`（加载DLL）。
      
    - **题目对应**：对应题目要求的 **“系统调用拦截”**，你需要捕获这些内核层的操作。
    
- **`entities`**
  
    - **含义**：行为涉及的具体对象。
      
    - **题目对应**：**“进程行为链分析”** 的核心数据。
      
        - `process_name` / `pid`: 当前进程。
          
        - `parent_process` / `parent_pid`: 父进程。通过这两组数据，后端图数据库可以使用 `Spawn` 关系构建进程树。
          
        - `command_line`: **最关键的取证数据**。例如 `powershell.exe -enc ...`，base64 编码的恶意指令就藏在这里。
	    - ==**当 `event_type` 为 `registry_set_value` 时：entities要补充**==

		- ==`registry_key`: 完整的注册表路径 (e.g., `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\EvilExe`)==
    
		- ==`registry_value_name`: 键名 (e.g., `EvilExe`)==
    
		- ==`registry_value_data`: 写入的数据 (e.g., `C:\Windows\Temp\trojan.exe`)==
		
		- ==（`file_create`/`file_modify`）和进程创建（`process_create`）的 `entities` 中增加 **HASH 值**。没有 Hash，无法在第（3）步中做“攻击者工具指纹特征匹配”。==
    
- **`behavior_features` (特征标记)**
  
    - **含义**：探针端（Agent）对该行为的初步判定结果。
      
    - **题目对应**：
      
        - `is_abnormal_parent`: true/false。对应 **“识别异常进程创建模式”**（例如 `word.exe` 启动了 `cmd.exe`，这是异常的）。
          
        - `has_memory_injection`: true/false。对应 **“内存行为分析”**（检测代码注入）。

##### 键值对详细解释：
- **`event_type`**
    - **含义**：流量事件的分类。
    - **重要值**：
        - `dns_tunnel_suspected`: 疑似 DNS 隧道。
          
        - `http_tunnel_suspected`: 疑似 HTTP 隐蔽信道。
          
        - `icmp_tunnel_suspected`: 疑似 ICMP 隧道。
        
    - **题目对应**：直接对应题目中的 **“隐蔽信道检测”** 要求。
- **`entities`**
  
    - **含义**：流量包中的元数据。
      
    - **题目对应**：
        - `domain`: 访问的域名。对应 **“C2 服务器的基础设施关联”**。
        - `payload_size`: 数据包大小。异常大的 TXT 记录往往意味着数据外传（Data Exfiltration）。
- **`traffic_features`**
  
    - **含义**：流量模型的分析结论。
      
    - **题目对应**：**“异常协议行为建模”**。
      
        - `is_covert_channel`: 是否为隐蔽信道。
          
        - `channel_type`: 具体的隧道类型（DNS/HTTP/ICMP）。

### 2. 《数据库设计字典 (Neo4j)》

[数据库设计字典](https://www.notion.so/2e6a6bed6ac7803c8d5ad3fd403b8937?pvs=21)

- **核心内容**：
    - **Elasticsearch Index 设计**：日志索引命名规则。
      
    - **Neo4j 图谱模型**：
      
        - **节点 (Nodes)**：定义有哪些实体（IP、进程、文件、域名）。
        
### 第一层：实体层 (Entities) —— 代表客观事实，这些数据直接来自采集探针
#### (1) `Process` (进程节点) —— **核心枢纽**

这是连接主机行为、文件操作和网络连接的最关键节点。
        
- **来源**：主机行为监控（进程创建）、主机日志（Sysmon/Audit）。
        - **唯一标识 (ID)**：`HostIP_PID_CreateTime` (仅用PID是不够的，因为PID会循环复用，必须加上创建时间戳)。
        - **关键属性**：
            - `pid`: 进程ID
            - `name`: 进程名 (e.g., `powershell.exe`)
            - `path`: 完整路径
            - `cmdline`: 命令行参数 (非常重要，用于检测恶意脚本)
            - `user`: 运行该进程的用户
            ==- port: 端口号==（新增）
        
#### (2) `File` (文件节点)

攻击者通常会落地文件（Webshell、木马）或窃取文件。
        
- **来源**：主机行为监控（文件操作）、主机日志。
        - **唯一标识 (ID)**：`HostIP_FilePath` (在同一台主机上路径唯一)。
        - **关键属性**：
            - `path`: 文件路径
            - `name`: 文件名
            - `hash`: MD5/SHA256 (用于威胁情报匹配)
            - `type`: 文件类型 (exe, dll, txt, php)
        
#### (3) `IP` (网络节点)

代表内网主机或外网攻击者 IP。
        
- **来源**：网络流量分析、主机日志（登录源IP）、主机行为（网络连接）。
        - **唯一标识 (ID)**：`IP_Address`。
        - **关键属性**：
            - `ip`: IP地址 string
            - `geo`: 地理位置 (如果是外网)
            - `type`: Internal (受害机) / External (攻击者) / Gateway
        
#### (4) `Domain` (域名节点)

用于识别 C2 服务器或钓鱼网站，特别是检测 DNS 隧道时需要。
        
- **来源**：网络流量分析（DNS请求、HTTP Host头）。
        - **唯一标识 (ID)**：`DomainName`。
        - **关键属性**：
            - `name`: 域名 (e.g., `malware.c2.com`)
            - `record_type`: A, CNAME, TXT
        - **关系 (Edges)**：定义实体间的动作（连接、创建、注入、解析）。
        

#### (5) `User` (用户节点)

用于分析权限提升和横向移动（基于账号的攻击）。
        
- **来源**：主机日志（登录/注销）。
        - **唯一标识 (ID)**：`HostIP_Username` (域环境下可以用 `Domain_Username`)。
        - **关键属性**：
            - `username`: 用户名
            - `sid`: 安全标识符
            - `group`: 所属组 (Admin, User)

#### （6）Registry（注册表操作）

- **来源**：主机行为监控（注册表操作）。
        - **唯一标识 (ID)**：`Registry_Key_Path` (字符串直接作为ID)。
        - **关键属性**：
			- `key`: 注册表键路径
			- `value_name`: 值名称
			- `value_data`: 对应的值数据 (关键取证点)
		

#### 第二层：检测层 (Detections) —— 代表分析结论，这些数据来自 ATT&CK 映射函数。

| **节点类型**          | **唯一标识 (ID Key)**  | **核心属性**                           | **来源**      |
| ----------------- | ------------------ | ---------------------------------- | ----------- |
| **`AttackEvent`** | `attack_id` (UUID) | `timestamp`, `confidence`, `stage` | ATT&CK 映射函数 |

#### 第三层：知识层 (Knowledge) —— 代表静态情报

这些数据来自 MITRE ATT&CK 知识库。

| **节点类型**        | **唯一标识 (ID Key)**            | **核心属性**         | **来源** |
| --------------- | ---------------------------- | ---------------- | ------ |
| **`Technique`** | `technique_id` (e.g., T1059) | `name`, `tactic` | 静态知识库  |


实体关系

| **源节点 (Source)** | **关系类型 (Type)**                         | **目标节点 (Target)** | **业务含义**                    | **数据来源**                  |
| ---------------- | --------------------------------------- | ----------------- | --------------------------- | ------------------------- |
| **Process**      | **`Spawn`**                             | **Process**       | 父子进程链，用于回溯攻击入口              | 主机行为 (ParentPID)          |
| **Process**      | **`Write`** / **`Read`** / **`Delete`** | **File**          | 文件操作，用于检测落地木马或窃取数据          | 主机行为 (FileOp)             |
| **Process**      | **`Load`**                              | **File**          | 加载 DLL/Image，用于检测劫持         | 主机行为 (ImageLoad)          |
| **Process**      | **`Inject`**                            | **Process**       | 内存注入，用于检测高级免杀               | 主机行为 (CreateRemoteThread) |
| **Process**      | **`Connect`**                           | **IP**            | **关键跨域关联**：进程发起的网络连接        | 主机行为 + 流量融合               |
| **Process**      | **`Resolve`**                           | **Domain**        | 域名解析，用于发现 DGA 或 C2 域名       | 网络流量 (DNS)                |
| **User**         | **`Logon`**                             | **IP** (Host)     | 用户登录主机，用于身份关联               | 主机日志 (Event 4624)         |
| **IP**           | **`Traffic_Flow`**                      | **IP**            | 纯网络层通信事实 (包含 Protocol/Size) | 网络流量                      |
| **Process**      | **`Write`**                             | **Registry**      | 修改注册表 (如添加启动项)              | 主机行为 (RegSet)             |


检测与知识层关系 (Detection Layers)

| **源节点**                         | **关系类型**         | **目标节点**        | **含义**                  |
| ------------------------------- | ---------------- | --------------- | ----------------------- |
| **Process** / **File** / **IP** | **`TRIGGERED`**  | **AttackEvent** | 某实体触发了具体的告警实例           |
| **AttackEvent**                 | **`IS_TYPE`**    | **Technique**   | 该告警属于某种标准的 ATT&CK 技术    |
| **AttackEvent**                 | **`NEXT_STAGE`** | **AttackEvent** | (可选) 时序上相邻的攻击步骤，用于构建攻击链 |
