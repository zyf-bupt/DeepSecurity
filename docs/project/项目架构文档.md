### 2. 核心模块详解

#### 📝 2.1 主机日志模块 (`utils/winlog/`)

**功能职责**:
- Windows Security/System 事件采集
- 日志范式化与关键字段提取
- 登录会话重建（4624/4634/4647 事件）
- 时间序列对齐（不同主机时钟同步）

**关键技术**:
- `win32evtlog` API 读取事件日志
- XML 解析提取结构化字段
- 事件去重（event_hash SHA256）
- 增量采集（state_store 断点续传）

**数据流**:
```
Windows Event Viewer
    ↓ (collector_windows.py)
解析 EventID/Level/Computer/Data
    ↓ (parser_winlogbeat.py)
范式化 JSON (result 字段)
    ↓ (hostlogs_ingest.py)
SQL Server.HostLogs 表
```

#### 🔍 2.2 主机行为监控模块 (`utils/behavior_monitor/`)

**功能职责**:
- 系统调用级监控（进程、文件、网络、注册表）
- 进程父子关系树构建
- 异常行为检测（代码注入、反射加载、持久化）

**平台支持**:
| 平台    | 工具      | 配置文件                 | 监控器                    |
|---------|-----------|-------------------------|--------------------------|
| Windows | Sysmon    | -                       | host_monitor_windows.py  |
| Linux   | Falco     | falco_rules.yaml        | host_monitor_linux.py    |

**数据示例**:
```json
{
  "timestamp": "2026-01-16T10:30:45Z",
  "event_type": "ProcessCreate",
  "process_name": "powershell.exe",
  "parent_process": "cmd.exe",
  "command_line": "powershell -enc <base64>",
  "user": "DESKTOP\\Admin",
  "suspicious_score": 0.85
}
```

#### 🌐 2.3 网络流量分析模块 (`utils/traffic_fenxi/`)

**功能职责**:
- 实时流量捕获（Scapy AsyncSniffer）
- 协议解析（TCP/UDP/HTTP/DNS/TLS）
- 会话重建（五元组关联）
- 隐蔽信道检测（DNS隧道、ICMP隧道、HTTP慢传输）

**采集模式**:
1. **在线抓包**: `live_capture.py` - Scapy 后台线程
2. **高性能抓包**: `live_capture_dumpcap.py` - Wireshark dumpcap
3. **离线分析**: `ingest_offline.py` - 导入 pcap 文件

**异常检测算法**:
- DNS 查询长度统计异常（熵值分析）
- ICMP 载荷大小异常
- HTTP 慢速传输检测（请求间隔分析）
- TLS 证书指纹黑名单匹配

#### 🎯 2.4 攻击溯源与关联分析模块 (`utils/trace/`)

**核心流程** (`main_pipeline.py`):
```python
while True:
    # 1. 增量拉取新数据（HostLogs/HostBehaviors/NetworkTraffic）
    new_data = db_loader.fetch_new_data(table, last_checkpoint)
    
    # 2. 构建知识图谱（Neo4j）
    graph_engine.ingest_host_behavior(new_data)
    graph_engine.ingest_network_traffic(new_data)
    
    # 3. ATT&CK 映射
    attacks = mapper.analyze_event(event)
    graph_engine.ingest_attack_events(attacks)
    
    # 4. 场景关联（同一攻击者的多个动作）
    scenario_linker.link_attack_scenario(time_window=300)
    
    # 5. APT 组织匹配
    apt_engine.match_apt_group(ttp_chain)
    
    # 6. 生成溯源报告（LLM 增强）
    llm_reporter.generate_report(scenario_id, evidence)
    
    # 7. 更新断点
    state_manager.update_checkpoint(table, new_id)
    
    sleep(10)  # 轮询间隔
```

**ATT&CK 映射规则** (`attack_rules.yaml`):
```yaml
- technique_id: T1059.001
  name: "PowerShell"
  pattern: "powershell.*-enc|-e |-nop|-w hidden"
  data_source: ["HostBehaviors", "HostLogs"]
  confidence: 0.85

- technique_id: T1071.004
  name: "DNS Tunneling"
  pattern: "dns_query_length > 50 && entropy > 4.5"
  data_source: ["NetworkTraffic"]
  confidence: 0.90
```

**知识图谱节点类型**:
- `Process`: 进程节点（pid, name, cmdline）
- `User`: 用户节点（username, domain）
- `IP`: IP 地址节点（src_ip, dst_ip）
- `File`: 文件节点（path, hash）
- `ATTACKTechnique`: ATT&CK 技术节点

**关系类型**:
- `SPAWNED`: 进程创建关系
- `ACCESSED`: 文件访问关系
- `CONNECTED_TO`: 网络连接关系
- `LOGGED_ON`: 用户登录关系
- `USES_TECHNIQUE`: 攻击技术使用关系

### 3. 数据库设计

#### SQL Server (`SecurityTraceDB`)

**HostLogs 表**:
```sql
CREATE TABLE dbo.HostLogs (
    id INT IDENTITY(1,1) PRIMARY KEY,
    result NVARCHAR(MAX),              -- 解析后的 JSON
    content NVARCHAR(MAX),             -- 原始日志
    create_time DATETIME2 DEFAULT GETDATE(),
    event_hash VARCHAR(64),            -- SHA256 去重
    host_name NVARCHAR(255),           -- 主机名
    CONSTRAINT UX_HostLogs_event_hash UNIQUE(event_hash)
);
```

**HostBehaviors 表**:
```sql
CREATE TABLE dbo.HostBehaviors (
    id INT IDENTITY(1,1) PRIMARY KEY,
    result NVARCHAR(MAX),              -- 行为分析结果
    content NVARCHAR(MAX),             -- 原始事件
    create_time DATETIME2 DEFAULT GETDATE()
);
```

**NetworkTraffic 表**:
```sql
CREATE TABLE dbo.NetworkTraffic (
    id INT IDENTITY(1,1) PRIMARY KEY,
    result NVARCHAR(MAX),              -- 流量解析结果
    content NVARCHAR(MAX),             -- 原始数据/元信息
    create_time DATETIME2 DEFAULT GETDATE()
);
```

**AttackReports 表**:
```sql
CREATE TABLE dbo.AttackReports (
    id INT IDENTITY(1,1) PRIMARY KEY,
    scenario_id VARCHAR(64),           -- 攻击场景唯一标识
    victim_ip VARCHAR(50),
    attacker_ip VARCHAR(50),
    confidence FLOAT,                  -- 置信度 0-1
    attribution_type VARCHAR(50),      -- 归因类型（APT/Script Kiddie）
    attribution_name NVARCHAR(255),    -- APT 组织名称
    start_time DATETIME2,
    end_time DATETIME2,
    report_json NVARCHAR(MAX),         -- LLM 生成的完整报告
    created_at DATETIME2 DEFAULT GETDATE()
);
```
