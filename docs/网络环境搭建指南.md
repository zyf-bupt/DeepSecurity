## 二、虚拟机配置

使用 **VMware Workstation** 或 **VirtualBox** 搭建 6 台虚拟机：

| 序号 | 主机名 | IP地址 | 操作系统 | 角色 | 安全区 |
|------|--------|--------|---------|------|--------|
| 1 | web01-dmz | 192.168.86.10 | Ubuntu 22.04 | Web服务器 | DMZ |
| 2 | srv-core | 192.168.86.130 | Ubuntu 22.04 | 核心服务器 | 内网核心 |
| 3 | dc-win | 192.168.86.131 | Windows Server 2022 | 域控制器 | 内网高安全 |
| 4 | jump-host | 192.168.86.132 | Ubuntu 22.04 | 跳板机 | 内网 |
| 5 | soc-node | 192.168.86.200 | Ubuntu 22.04 | SOC节点 | 安全管理 |
| 6 | attacker | 45.33.22.11 | Kali Linux | C2服务器 | 外部 |

## 三、VMware 网络配置

### 步骤1：创建虚拟网络
```
1. VMware → 编辑 → 虚拟网络编辑器
2. 添加网络：
   - VMnet2 (DMZ):     192.168.86.0/24，桥接模式
   - VMnet3 (内网):     192.168.86.0/24，仅主机模式
   - VMnet4 (管理区):   192.168.86.0/24，仅主机模式
```

### 步骤2：配置虚拟机网络适配器
每台虚拟机分配对应的 VMnet 网络适配器。

### 步骤3：配置防火墙规则（示例）
```powershell
# Windows 域控制器 — 限制入站
New-NetFirewallRule -DisplayName "Block DMZ to Internal" -Direction Inbound -RemoteAddress 192.168.86.10 -Action Block

# Linux — iptables
sudo iptables -A INPUT -s 192.168.86.10 -j DROP  # DMZ不能直接访问内网
sudo iptables -A INPUT -s 45.33.22.11 -j DROP     # 外部IP不能访问
```

## 四、SQL Server 配置

系统使用 SQL Server 作为持久化存储：

```powershell
# 连接信息
服务器: <SERVER_IP>,1433
数据库: SecurityTraceDB
用户: sa
密码: <DB_PASSWORD>
```

建表脚本：`knowledge_base/create_database.sql`

```powershell
sqlcmd -S <SERVER_IP>,1433 -U <DB_USERNAME> -P <DB_PASSWORD> -d SecurityTraceDB -i knowledge_base/create_database.sql
```

## 六、快速启动

```powershell
# 1. 安装 Python 依赖
pip install -r requirements.txt

# 2. 安装前端依赖
cd frontend && npm install

# 3. 启动系统（自动启动 Vite + Flask）
python app_launcher.py

# 4. 浏览器打开
http://localhost:5173
```
