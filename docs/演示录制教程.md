# SecTrace 演示录制教程

## 环境准备

### 需要打开的程序
1. **终端1（管理员PowerShell）**：Flask + Vite 系统
2. **终端2（管理员PowerShell）**：行为监控 Agent
3. **浏览器**：Chrome/Edge 打开 `http://localhost:5173`
4. **录屏软件**：OBS Studio（免费）或 Windows 自带 Xbox Game Bar（Win+G）

---

## 录制流程（预计 8-12 分钟）

### 第1段：系统总览（1分钟）
```
1. 浏览器打开 http://localhost:5173
2. 展示首页 — 系统模块、分析流程、快速入口
3. OBS 录制全屏或浏览器窗口
4. 配音："这是基于大模型的攻击行为全方位检测、捕获与溯源系统..."
```

### 第2段：网络拓扑（1分钟）
```
1. 点击左侧「网络拓扑」
2. 展示 6 节点、5 安全区的 Canvas 拓扑图
3. 鼠标悬停在节点上显示详情
4. 点击节点查看右侧详细信息
5. 配音："系统模拟了企业网络环境，包含 DMZ、内网、高安全区共 6 个节点..."
```

### 第3段：场景管理 + 检测（2分钟）
```
1. 点击「场景管理」
2. 点击 APT 场景「启动」→ 观察进度条和事件流
3. 切换到「攻击检测」→ 观察实时告警流
4. 展示 RAG 知识库搜索（输入 "mimikatz" 搜索）
5. 配音："系统内置两种攻击场景，也可通过 behavior agent 采集真实数据..."
```

### 第4段：行为监控 + 真实攻击演示（2分钟）
```
1. 切到终端2（管理员PowerShell），展示 agent 正在运行
   python utils/behavior_monitor/client_agent.py --server http://<SERVER_IP>:5000 --verbose
2. 终端3（管理员）执行演示攻击脚本：
   python utils/demo_attack_script.py
3. 展示 agent 终端捕获到的 process_create、dns_query、network_connection 等事件
4. 切回浏览器「行为监控」页面 → 刷新 → 展示新入库的行为数据
5. 配音："刚才执行的脚本产生了真实的系统行为，agent 实时捕获并上报..."
```

### 第5段：多源LLM分析 + 报告（2分钟）
```
1. 点击「攻击链分析」
2. 填入时间范围（如 2026-07-07T15:00 ~ 留空）
3. 点击「🤖 多源LLM分析」
4. 等待 5-10 秒（展示"分析中"状态）
5. 分析完成 → 左侧出现新报告
6. 点击报告 → 展示三个Tab：
   - 拓扑概览（SVG攻击链图）
   - LLM分析（DeepSeek 中文威胁评估报告）
   - 报告详情（JSON原始数据）
7. 配音："系统同时读取主机日志、行为数据、网络流量三源数据，
   经过 ATT&CK 规则匹配后，调用 DeepSeek 大模型生成结构化威胁分析报告..."
```

### 第6段：溯源分析 + 归因（1分钟）
```
1. 点击「溯源分析」→ 自动加载已有报告
2. 点击报告 → 展示攻击路径、IOCs、AI研判报告
3. 点击「溯源归因」→ 时间范围 → 🤖多源LLM归因分析
4. 展示 DeepSeek 归因结果
5. 配音："溯源模块基于多智能体协作框架验证攻击链，结合威胁情报进行归因..."
```

### 第7段：流量分析 + 日志分析（1分钟）
```
1. 点击「流量分析」
2. 展示 dumpcap 自动检测到的网卡列表
3. 选择 WLAN 网卡 → 开始捕获 → 停止并导入
4. 展示流量列表（按时间/IP/event_type筛选）
5. 点击「日志分析」→ 展示 HostLogs 列表
6. 配音："系统支持真实网络流量采集和 Windows 事件日志归一化..."
```

## OBS 录屏设置

1. **下载 OBS**：https://obsproject.com/
2. **添加源**：显示器采集（全屏）或 窗口采集（浏览器）
3. **音频**：麦克风（配音） + 桌面音频（系统声音，可选）
4. **输出设置**：
   - 输出模式：简单
   - 录像质量：高质量
   - 录像格式：MP4
   - 编码器：硬件（NVENC/AMD）或 软件 x264
5. **开始录制**：点击「开始录制」
6. **停止录制**：点击「停止录制」→ 文件保存在「视频」文件夹
